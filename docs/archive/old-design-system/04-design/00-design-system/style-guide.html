Dark Mode System

Executive Dark Mode

SuperSwift's dark mode maintains the sophisticated, executive-level aesthetic while providing comfortable viewing in low-light environments.

Dark Mode Principles

  • Executive Elegance: Maintains professional sophistication
  • Enhanced Contrast: Improves readability and accessibility
  • Preserved Branding: Gold accent remains prominent
  • Strategic Intelligence: Enhanced Trust and Success colors

💡 Use the 3-mode theme toggle in the header: System, Light, Dark

Light vs Dark Mode Colors

Light Mode

Foreground: Deep Black
Accent: Sophisticated Gold
Trust: Professional Blue

Dark Mode

Background: Rich Dark

Implementation Guide

/* 3-Mode Theme System Implementation */
.dark {
  --executive-background: oklch(0.1200 0 0);    /* Rich Dark */
  --executive-foreground: oklch(0.9850 0 0);    /* Clean White */
  --executive-trust: oklch(0.6500 0.1400 240.0000);  /* Enhanced Blue */
  --executive-success: oklch(0.7200 0.1600 160.0000); /* Enhanced Green */
}

/* JavaScript 3-Mode Toggle */
function applyTheme(theme) {
  if (theme === 'system') {
    const systemPreference = window.matchMedia('(prefers-color-scheme: dark)').matches;
    document.documentElement.classList.toggle('dark', systemPreference);
  } else {
    document.documentElement.classList.toggle('dark', theme === 'dark');
  }
  localStorage.setItem('theme', theme);
}

Logo Usage Guidelines

Do's

  • Use icon logo in 16px, 24px, 32px, 48px sizes
  • Place on executive-accent background with padding
  • Maintain aspect ratio when scaling
  • Use rounded corners (8px radius for containers)

Don'ts

  • Don't stretch or distort the logo
  • Don't use on low-contrast backgrounds
  • Don't add effects, shadows, or outlines
  • Don't use without proper padding

Usage Guidelines

Component Usage

Generic Components

  • Navigation: Use for all app navigation (header, sidebar, breadcrumbs)
  • Buttons: Use for all user actions (primary, secondary, accent based on importance)
  • Forms: Use for all data input (auth, settings, user preferences)
  • Cards: Use for content containers (dashboards, lists, previews)

Assessment-Specific Components

  • Progress: Use only in assessment flows to show completion
  • Phase Indicators: Use to show assessment phase progression
  • Validation: Use for AI analysis and confirmation screens
  • Input Methods: Use for multi-modal assessment inputs

Accessibility Guidelines

Touch Targets

  • • Minimum 44px height for mobile touch targets
  • • 48px minimum for primary interactive elements
  • • Adequate spacing between clickable elements

Color Contrast

  • • WCAG AA compliance for all text combinations
  • • High contrast focus indicators
  • • Semantic color coding with additional indicators
